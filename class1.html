History of the Internet

In 1957, the first satellite was launched into orbit.

In 1957, Russia launches the very first satellite into space. This set off the "race for space" as Americans realized that while we had been focused on building biggers cars and TVs, other countries were exploring opportunities in space.
  • 1957 - First satellite in space
  • 1958 - ARPA is formed
  • 1969 - ARPAnet sends it's first message
  • 1991 - The World Wide Web is created
Not wanting to be left behind, the U.S. government created a division called ARPA (Advanced Research Projects Agency). This agency focused on creating space-age technologies like rockets, weapons and computers. In the 1960s - ARPA began creating a network of computers that could talk to each other, known as ARPAnet. In 1960 - ARPAnet sent it's very first message, "LOGIN". This was the beginnings of what we now call the Internet. In 1991 - Tim Berners-Lee, a computer programmer in Switzerland, had the idea "computers are now able to send information, but what if they could also request information?" With this idea, Tim published the first website.

Requesting a website

Browser -> ISP with DNS Access -> Router -> Google

Google -> Router -> ISP -> Browser

Materials: - Browser Material - Piece of paper that reads: From: 65.30.200.101 (user) To: www.google.com Message: Homepage - DNS Material - DNS List www.boston.gov - 107.154.103.58 www.google.com - 172.217.11.46 - ISP sends paper to DNS, DNS adds IP Address, back to Browser - Piece of paper that reads: From: 74.125.19.104 (www.google.com) To: 65.30.200.101 (user) Message: Homepage - Google Server - Envelope - Paper that says "html" - Paper that says "css" Steps: - Browser passes paper to ISP - ISP passes paper to DNS - DNS adds the IP Address - DNS passes back to ISP - ISP passes off to the router - Router sends paper to Google server - Google server creates an envelope with To and From address - Google server adds HTML and CSS files to envelope - Google server passes envelope to router - Router sends envelope to ISP - ISP passes the envelope to the browser - Browser renders the html and css files When I enter an website's address in the browser's address bar and press enter, my computer is going to send a request to the computer's Internet Service Provider. My request includes three pieces of information, my IP address, the name of the website I'm requesting and any additional information about my request. The Internet Service Provider is going to look at the website name and using something called DNS to translate the website name to an IP address. Just like my computer has an IP address, all other computers have an IP address. The reason we type in a website's name instead of an IP address is because they're very hard to remember. For example, I can go to Gooogle by typing in their IP address (demo: ping google.com and copy IP address). This is similar to how we store someone's name when adding them to our contacts. We don't want to rememeber their phone number because it's easier for us to remember their name. So the Internet Service Provider knows the website's name but the internet communicates with IP Addresses, not by website names. So before the ISP can send the request, they need to know the website's IP Address. To do this, the ISP can ask the DNS system for help. The DNS system will look up the website name and find the IP address. The Internet Service Provider will then use their access to the internet to pass on the request. The Internet will use information that's in the request to identify where the information lives and return a response. That response will include two files, an html file and a CSS file. Internet connections are established by delivering electronic pulses, lightwaves in fiber optic cables and radiowaves.

Building a Website

The browser expects to receive two important files, an HTML file and a CSS file.

Image from codercv.com

The HTML file contains all the content for our site. The CSS file is used to make our website beautiful. CSS is how we add color to our page and place each picture or link in a specific place. We will talk about CSS later on in class. First, let's talk about HTML.

What is HTML?

HTML stands for: Hypertext Markup Language

Hypertext - refers to links on a webpage

Markup - informs the browser how to present text

Language - set of rules so others can understand

HTML is called a "markup" language because we use it to inform the browser of how to treat text differently.

Why do we use HTML?

HTML allows us to tell the browser important information about our content.

Image from Markup by ShowMe

Similar to how a teacher markups a student's report to give information to the student, we want to "markup" our text so the browser knows how to show our content

How do we write HTML?

Image from internetingishard.com
We write HTML by using something called "tags". To "markup" our content with HTML, we create an opening tag, add our content, and then add a closing tag. You can think of HTML tags as bookends. You need both bookends to tell the browser where to start and where to end. Once we have an opening tag with content and a closing tag, we've created what's called an HTML element.
