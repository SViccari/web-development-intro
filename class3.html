All the HTML elements that we've been working with have a default type of box. The h1 and p elements are block boxes while em and strong elements are inline boxes.

Block vs Inline Elements

Replace existing CSS with the following:

            
              h1, p {
                background-color: lightgrey;
              }

              em, strong {
                background-color: lightblue;
              }

Block vs Inline Elements

This example demonstrates:

  • Block elements always appear below the previous block element.
  • The width of block elements is set automatically to the parent container. In this case, the browser window.
  • The height of block elements is based on the content it contains. The height adjusts accordingly when the browser window is narrowed.
  • The width of inline elements is based on the content it contains, not the width of the parent element.

Changing Box Behavior

We can override the default box type of HTML elements with the CSS display property. For example, if we want to make an inline element behave like a block element. 

            
              em {
                display: block;
              }

CSS Box Model

To understand how CSS treats each HTML element as a "box", we'll examine the CSS Box Model - a set of rules that determine the dimensions of every element on a web page.

Image from internetingishard.com

The box model gives each element (both block and inline) four properties: content, padding, border, and margin.

Inspecting the CSS Box Model

To help us along the way, we'll use a helpful browser tool called the inspector To open the inspector, right-click on your website and select "inspect".

CSS Box Model - Content

Content - the text, image or media content in the element.

We've already worked with content, so on to padding!

CSS Box Model - Padding

Padding - the space between the box's content and its border.


Let's test it out by adding padding to the header element:

            
              h1 {
                padding: 50px;
              }
This adds 50 pixels to each side of the header element. Notice how the background color expands to fill this space. That's always the case for paddings because it's inside the border and everything inside the border gets a background.

CSS Box Model - Padding

Add padding to a specific side of the element:

            
              h1 {
                padding-top: 20px;
                padding-bottom: 20px;
                padding-left: 10px;
                padding-right: 10px;
              }

CSS Box Model - Padding

Padding shorthand format: vertical horizontal

Image from internetingishard.com

            
              h1 {
                padding: 20px 10px;
              }
Typing out all these properties can be tiresome, so CSS provdes an alternative shorthand form of the padding property. This shorthand lets us set the top/bottom and left/right padding with only one line of CSS.

CSS Box Model - Padding

Padding shorthand format: top right bottom left

Image from internetingishard.com

            
              h1 {
                padding: 20px 10px 20px 10px;
              }

CSS Box Model - Borders

Border - The line drawn around the content and padding of an element.

Image from internetingishard.com

            
              h1 {
                border: 1px solid grey;
              }
This tells the browser to draw a thin gray line around our heading. Notice how the border bumps right up next to the padding with no space in between. And, if you shrink your browser enough for the heading to be split over two lines, both the padding and the border will still be there.

CSS Box Model - Borders

Like padding, there are -top, -bottom, -left, -right, variants for the border property 

            
              h1 {
                border-top: 1px solid pink;
                border-right: 1px solid red;
                border-bottom: 1px solid grey;
                border-left: 1px solid green;
              }
Borders are great for debugging!

CSS Box Model - Margin

Margin - define the space outside of the element's border. Or, rather, the space between a box and its surrounding boxes.

Add some space (margin) to the bottom of each paragraph:

            
              p {
                margin-bottom: 50px;
              }
This demonstrates a side-specific variant of the margin property but this also accepts the same shorthand formats as padding.
